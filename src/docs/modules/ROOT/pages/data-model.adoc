MyMediaScanner stores all collection data locally in SQLite via the Drift ORM.
The schema is versioned and migrated automatically on startup.
This page documents schema version {schema-version}.

== Schema Overview

Schema version {schema-version} defines 13 tables:

[cols="1,2", options="header"]
|===
| Table | Purpose

| `media_items`
| Central catalogue of scanned physical media

| `tags`
| User-defined labels for categorising items

| `media_item_tags`
| Many-to-many join between items and tags

| `shelves`
| Named collections for organising items

| `shelf_items`
| Many-to-many join with display ordering

| `barcode_cache`
| Raw API responses cached per barcode

| `sync_log`
| Audit trail for all push/pull sync operations

| `borrowers`
| People who may borrow items

| `loans`
| Records of items lent to borrowers

| `rip_albums`
| FLAC rip directories linked to physical items

| `rip_tracks`
| Individual FLAC tracks within a rip album

| `batch_sessions`
| Groups of barcodes scanned in batch mode

| `batch_queue_items`
| Individual queue entries within a batch session
|===

== Table Reference

=== media_items

The central table.
Every physical media item scanned into the catalogue has a row here.

[cols="2,1,4", options="header"]
|===
| Column | Type | Description

| `id`
| TEXT PK
| UUID v4, generated by the client

| `barcode`
| TEXT
| Raw barcode string as scanned

| `barcode_type`
| TEXT
| One of: `ean13`, `upca`, `isbn10`, `isbn13`, `imdb`, `unknown`

| `media_type`
| TEXT
| One of: `film`, `tv`, `music`, `book`, `game`, `unknown`

| `title`
| TEXT
| Primary title

| `subtitle`
| TEXT
| Subtitle or series qualifier (nullable)

| `description`
| TEXT
| Long-form synopsis or description (nullable)

| `cover_url`
| TEXT
| Remote URL for cover art (nullable)

| `year`
| INTEGER
| Release year (nullable)

| `publisher`
| TEXT
| Publisher, label, or studio (nullable)

| `format`
| TEXT
| Physical format, e.g. `Blu-ray`, `CD`, `Hardback` (nullable)

| `genres`
| TEXT
| JSON array of genre strings, e.g. `["Drama","Thriller"]`

| `extra_metadata`
| TEXT
| JSON object for API-specific fields not mapped to dedicated columns

| `source_apis`
| TEXT
| JSON array of API names that contributed data, e.g. `["tmdb","upcitemdb"]`

| `user_rating`
| REAL
| User rating from 1.0 to 5.0 (nullable)

| `user_review`
| TEXT
| Free-text review written by the user (nullable)

| `date_added`
| INTEGER
| Unix epoch (ms) when the row was first inserted

| `date_scanned`
| INTEGER
| Unix epoch (ms) when the barcode was physically scanned (nullable)

| `updated_at`
| INTEGER
| Unix epoch (ms) of the last local modification; used for sync conflict resolution

| `synced_at`
| INTEGER
| Unix epoch (ms) of the last successful sync to PostgreSQL (nullable)

| `deleted`
| INTEGER
| Soft-delete flag: `0` = active, `1` = deleted
|===

NOTE: All JSON columns are stored as TEXT in SQLite.
The Drift DAOs handle serialisation and deserialisation.

=== media_item_tags

Join table linking media items to tags.
A row exists for each active tag applied to an item.

[cols="2,1,4", options="header"]
|===
| Column | Type | Description

| `media_item_id`
| TEXT FK
| References `media_items.id`

| `tag_id`
| TEXT FK
| References `tags.id`
|===

The composite `(media_item_id, tag_id)` is the primary key.

=== barcode_cache

[cols="2,1,4", options="header"]
|===

| TEXT PK

| `media_type_hint`
| TEXT

| `response_json`
| TEXT
| Raw JSON body returned by the source API

| `source_api`
| TEXT

=== sync_log

Provides a full audit trail of every push and pull operation.
Each row represents one entity being synchronised in one direction.

[cols="2,1,4", options="header"]
|===
| Column | Type | Description

| `id`
| TEXT PK
| UUID v4

| `entity_type`
| TEXT
| Table name of the entity, e.g. `media_items`, `loans`

| `entity_id`
| TEXT
| UUID of the entity that was synchronised

| `operation`
| TEXT
| One of: `insert`, `update`, `delete`

| `payload_json`
| TEXT
| JSON snapshot of the row at the time of the operation

| `created_at`
| INTEGER
| Unix epoch (ms) when the log entry was created locally

| `attempted_at`
| INTEGER
| Unix epoch (ms) of the most recent sync attempt (nullable)

| `synced`
| INTEGER
| `1` if the operation completed successfully, `0` otherwise

| `error_message`
| TEXT

| `duration_ms`
| INTEGER

| TEXT

| `resolved_by`
| TEXT
| Conflict resolution outcome: `auto` (last-write-wins) or `user` (manual resolution)

== PostgreSQL Differences

When data is synced to the self-hosted PostgreSQL instance, the schema diverges slightly from the SQLite representation:

[cols="1,2,2", options="header"]
|===
| Aspect | SQLite (Drift) | PostgreSQL

| Primary key type
| `TEXT` storing UUID strings
| Native `UUID` type

| Timestamps
| `INTEGER` (Unix epoch ms)
| `TIMESTAMPTZ` (timezone-aware)

| Device identifier
| Not present
| `device_id TEXT NOT NULL` column on each syncable table; identifies which client last wrote the row

| Boolean / flag columns
| `INTEGER` (`0` / `1`)
| Native `BOOLEAN`
|===

The `device_id` column on the PostgreSQL side enables per-device uniqueness constraints and simplifies debugging of sync conflicts when multiple devices contribute data.

== Soft Delete Convention

MyMediaScanner never hard-deletes rows from any table.
Instead, the `deleted` column is set to `1`.

This convention is applied consistently across all tables that support user-managed data:
`media_items`, `tags`, `shelves`, `borrowers`, `loans`, `rip_albums`, and `rip_tracks`.

The benefits are:

* Sync correctness — deleted records must propagate to remote devices so they are not re-created during a pull.
* Audit trail — the full history of the collection is preserved.
* Undo capability — a future feature can restore soft-deleted items without any data loss.

All DAOs filter out deleted rows by default using a `WHERE deleted = 0` clause.
Sync queries explicitly include deleted rows so that deletions are replicated correctly.

CAUTION: Never issue a `DELETE` statement directly against the database.
Always set `deleted = 1` and update `updated_at` so the change is captured by the sync engine.
