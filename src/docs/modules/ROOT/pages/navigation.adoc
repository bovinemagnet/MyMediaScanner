MyMediaScanner uses *GoRouter* with a `StatefulShellRoute` to provide a persistent navigation shell across all platforms.
The shell adapts its chrome to the host platform: a glassmorphism bottom navigation bar on mobile and a desktop sidebar rail on macOS, Windows, and Linux.

== Platform-Adaptive UI

GoRouter's shell route renders through `AppScaffold`, which inspects the current platform at build time via `core/utils/platform_utils.dart` and selects the appropriate navigation chrome.

=== Android and iOS

* *Bottom navigation bar* with four visible destinations: Home, Library, Scan, and Insights.
* A floating action button (FAB) provides a quick-scan shortcut from the Dashboard.
* The scanner screen renders a full-screen camera overlay using `mobile_scanner` (ML Kit).
* Shelves, Batch Editor, Settings, and Rips are accessible via AppBar actions or secondary navigation, not the bottom bar.

=== macOS, Windows, and Linux

* *Navigation rail / sidebar* shows all 12 branches, including the Scanner entry.
Desktop-only branches (Rips, Wishlist, Locations, Series, Wishlist Suggestions) are hidden on mobile builds.
At window widths below 600 pixels the sidebar collapses into a drawer accessible via a hamburger menu button.
* The Scanner screen hosts a focused text field accepting keyboard-wedge and USB barcode scanner input, plus a toggle to switch to webcam-based scanning.
Webcam scanning is available on all desktop platforms — macOS via `mobile_scanner`, Windows and Linux via `camera_desktop` with `flutter_zxing`.
* The Collection and Shelves screens use `MasterDetailLayout` with a draggable divider between the master and detail panes.
The split ratio is persisted to SharedPreferences.
* `TableKeyboardNavigation` provides arrow-key, Home/End, Enter, and Delete navigation for the collection and rips table views.
* Desktop screens use `ScreenHeader` instead of `AppBar`.
`AppBar` is reserved for mobile screens that require back-navigation affordance.

=== Material 3 Breakpoints

The layout system respects the following Material 3 window size class breakpoints.

[cols="2,1,3",options="header"]
|===
|Class
|Width
|Layout Behaviour

|Compact
|< 600 px
|Single-column; bottom navigation bar; no sidebar.

|Medium
|600 – 1 199 px
|Two-column where applicable; navigation rail (collapsed).

|Expanded
|≥ 1 200 px
|Full sidebar with labels; master–detail layout for Collection.
|===

TIP: The `MasterDetailLayout` widget used on Collection and Item Detail screens switches automatically between single-pane and dual-pane modes based on the current breakpoint.
See xref:architecture.adoc#state-management[State Management] for the provider that drives the selection state.
